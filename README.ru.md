# LOCANEX

Браузерное расширение для GeoGuessr — перехватывает координаты из игрового API и отображает их в тёмном информационном окне на ПК или в браузере телефона.

- **PC Popup**: плавающая панель с картой, координатами, погодой, Wikipedia, данными о стране
- **Mobile Relay**: WebSocket-сервер транслирует данные игры на телефон в реальном времени
- **Автопереключение**: попап скрывается, когда телефон подключён, и появляется снова при отключении

## Архитектура

```
GeoGuessr page
     │
     ├── xhr_inject.js (MAIN world, перехватывает XHR/fetch)
     │        │
     │        ▼
     │   content.js (MAIN world, парсит координаты, собирает HTML)
     │        │
     │        ├──► bridge.js (isolated world, пересылает событие ov-render)
     │        │              │
     │        │              ▼
     │        │         background.js (service worker)
     │        │              │
     │        │              ├──► panel.html + panel.js + panel-render.js (PC попап)
     │        │              │
     │        │              └──► server.js (WebSocket relay)
     │        │                         │
     │        │                         └──► Браузер телефона (SHELL-страница)
     │        │
     │        └──► settings.html + settings.js (страница настроек)
     │
     └──► Nominatim, Open-Meteo, Wikipedia, REST Countries (сторонние API)
```

### `background.js` — Service Worker

Центральный маршрутизатор сообщений, менеджер попапа, WebSocket-клиент.

| Функция | Назначение | Параметры | Поведение |
|---------|-----------|-----------|-----------|
| `tryRender()` | Отобразить кешированный HTML в попапе | нет | Если есть порт панели — шлёт через `port.postMessage`. Если ID окна есть в хранилище, но окна нет — открывает новое. Иначе создаёт окно. Защита от двойного открытия через флаг `isOpening`. |
| `openPanel()` | Создать окно попапа | нет | Создаёт всплывающее окно 360x780 (нефокусированное). Сохраняет `panelWinId` в `chrome.storage.session`. Таймаут безопасности 3 секунды. |
| `getWsUrl()` | Собрать URL WebSocket из настроек | нет | Нормализует `serverUrl`: заменяет `http://` на `ws://`, убирает `/relay`. По умолчанию: `ws://localhost:3000`. |
| `connectWS()` | Подключиться к relay-серверу | нет | Создаёт WebSocket к `getWsUrl() + '/ws'`. При открытии: `wsConnected=true`, оповещает настройки, отправляет кешированный HTML. При сообщении: парсит JSON, обрабатывает `phone_count` — закрывает попап, если телефон подключён. При закрытии: сбрасывает состояние, авто-переподключение каждые 3 с. |
| `disconnectWS()` | Разорвать WebSocket | нет | Отменяет таймер переподключения, закрывает сокет, сбрасывает `wsConnected`/`phoneConnected`. |
| `wsSend(html)` | Отправить HTML через WS | `html` (строка) | Сериализует `{type:'render',html}` в JSON. Если сокета нет — вызывает `connectWS()`. |
| `notifySettingsPage(data)` | Обновить статус в настройках | `data` (объект) | Шлёт `ws_status_update` через `chrome.runtime.sendMessage`. Беззвучно ловит ошибки, если получатель не найден. |

**Обработчики сообщений** (через `chrome.runtime.onMessage`):

| Тип | Отправитель | Действие |
|-----|------------|----------|
| `render` | `bridge.js` | Кеширует HTML в `chrome.storage.session`, пересылает в WS если relay включён, вызывает `tryRender()` если попап включён и телефон не подключён. |
| `get_settings` | страница настроек | Возвращает текущие `settings`, `wsConnected`, `phoneConnected`. |
| `save_settings` | страница настроек | Сохраняет новые настройки. Включает/выключает WebSocket в зависимости от `mobileRelay`. |
| `ws_status` | любой | Возвращает `{wsConnected, phoneConnected}`. |

**События жизненного цикла**:
- `chrome.runtime.onConnect` — захватывает порт панели, сразу отправляет кешированный HTML, сбрасывает при отключении.
- `chrome.windows.onRemoved` — сбрасывает `panelWinId`/`panelPort`/`isOpening` при закрытии попапа.
- `chrome.storage.session.get` при старте — восстанавливает `ov_lastHtml` и `ov_panelWinId`.
- `chrome.storage.local.get` при старте — восстанавливает настройки; авто-подключает WS, если relay был включён.

### `content.js` — MAIN World Script

Внедряется в страницы GeoGuessr. Перехватывает API-вызовы, парсит координаты, обогащает данными из внешних API, собирает HTML для попапа.

| Функция | Назначение | Параметры | Поведение |
|---------|-----------|-----------|-----------|
| `fetchAndInjectScript(url)` | Динамически загрузить JS-файл | `url` (строка) | Загружает `url` через fetch, создаёт `<script>`, вставляет в head, самоудаляется при загрузке. |
| `isDecimal(str)` | Проверить, является ли строка десятичным числом | любое значение | Возвращает true, если строка содержит точку и парсится в конечное число. |
| `haversine(lat1, lon1, lat2, lon2)` | Расстояние по дуге большого круга | две пары lat/lon | Возвращает расстояние в км по формуле гаверсинуса. |
| `countryToFlag(code)` | Флаг из ISO-кода | 2-символьный код | Конвертирует ISO 3166-1 alpha-2 в региональные индикаторы Unicode. |
| `toDMS(decimal, isLat)` | Десятичные в градусы/минуты/секунды | число, флаг широты | Возвращает формат `DD°MM′SS.s″ N/S/E/W`. |
| `playBeep()` | Звуковое оповещение | нет | Web Audio API: синусоида 880→440 Гц, 300 мс, громкость 0.15. |
| `getHistory()` | Прочитать историю локаций | нет | Возвращает массив из `localStorage.ov_history` (макс. 5). |
| `saveToHistory(data)` | Добавить в историю | объект с lat/lng/city/country/ts | Добавляет в начало, обрезает до 5, пишет в `localStorage`. |
| `getTimezone(lat, lng)` | Вычислить часовой пояс | широта/долгота | Рассчитывает по долготе (15 градусов = 1 час) относительно смещения UTC браузера. Возвращает `{name, time}`. |
| `getDrivingSide(code)` | Сторона движения | код страны | Возвращает `'LHT'` или `'RHT'` из статического словаря 51 страны. |
| `getPhoneCode(code)` | Телефонный код страны | код страны | Возвращает `+XX` из словаря 150+ стран. |
| `getDomain(code)` | Национальный домен | код страны | Возвращает `.xx`. Если нет в словаре — нижний регистр кода. |
| `getLanguage(code)` | Распространённые фразы | код страны | Возвращает 4 примера приветствий/указателей на местном языке (80+ стран). |
| `getEmergency(code)` | Номера экстренных служб | код страны | Возвращает номера полиции/скорой/пожарных через запятую. |
| `getAlphabet(code)` | Алфавит страны | код страны | Возвращает символы алфавита/письменности (60+ стран). |
| `getGeoTipsUrl(code)` | Ссылка на GeoTips | код страны | Возвращает URL с регионом из статического словаря (europe/asia/...). |
| `getWikipedia(city, country)` | Статья Wikipedia | город и/или страна | Запрашивает summary через REST API Wikipedia, возвращает desktop URL. |
| `getClimate(lat, lng)` | Текущая погода | широта/долгота | Open-Meteo API. Возвращает `{temp, humidity, precip, weather}`. 20+ WMO-кодов. |
| `getPopulation(country)` | Население страны | название страны | REST Countries API. Форматирует: 1.2B / 67.3M / 850K. |
| `getCoordInfo()` | Обратное геокодирование | нет (использует `lat`/`long`) | Nominatim API с `accept-language=en`. Возвращает OSM-объект. |
| `buildData(info)` | Собрать все обогащённые данные | результат Nominatim | Координирует запросы к Wikipedia, погоде, населению. Возвращает полный объект данных. |
| `buildHTML(data)` | Сгенерировать HTML | объект данных | Производит полный HTML-документ со встроенным CSS, JSON-данными и ссылками на скрипты. |
| `openOrFocusPopup(html)` | Отправить событие рендера | строка HTML | Инициирует `CustomEvent('ov-render', {detail: html})`. Ловится `bridge.js`. |
| `updatePopup()` | Полный конвейер обновления | нет (использует `lat`/`long`) | `getCoordInfo()` → `buildData()` → `buildHTML()` → `openOrFocusPopup()`. |

**Обработчик сообщений** (`window.addEventListener('message', ...)`):

Принимает перехваченные XHR/fetch данные от `xhr_inject.js`:
- Парсит JSON-массивы из ответа GeoGuessr API
- Извлекает координаты из двух возможных уровней вложенности
- Валидирует диапазоны (-90..90, -180..180)
- Дубликаты отсекаются через `lastProcessedLat`/`lastProcessedLng`
- При новом раунде: `playBeep()` + `updatePopup()`

**Опрос** (`setInterval`, каждые 2 с):

Отслеживает конец раунда по наличию/отсутствию DOM-элементов GeoGuessr. При выходе из игры сбрасывает состояние.

**Горячая клавиша**:

`Ctrl+Shift+R` — принудительно вызывает `updatePopup()`, если координаты есть, но попап не показался.

### `bridge.js` — Прослойка isolated world

Минимальный скрипт в isolated world. Реле для передачи событий `ov-render` в service worker.

| Функция | Назначение | Поведение |
|---------|-----------|-----------|
| обработчик `ov-render` | Переслать рендер | Слушает `document.addEventListener('ov-render')`, вызывает `chrome.runtime.sendMessage({type:'render', html})`. |

Файл нужен, потому что Chrome требует, чтобы сообщения в service worker отправлялись из isolated world, а `content.js` работает в MAIN world и не имеет доступа к `chrome.runtime`.

---

### `xhr_inject.js` — Перехватчик XHR/fetch

Внедряется в MAIN world страниц GeoGuessr. Модифицирует `XMLHttpRequest` и `window.fetch`.

| Перехват | Метод | Поведение |
|----------|-------|-----------|
| `XMLHttpRequest.open/send` | Оverride прототипа | Добавляет обработчик `load`, который отправляет `{type:'xhr', data: responseText}` в window. |
| `window.fetch` | Обёртка | Клонирует ответ, читает как текст, отправляет `{type:'fetch', data}` в window. Возвращает оригинальный ответ. |

Все данные возвращаются в `content.js` через `window.postMessage`.

### `panel.html` — Оболочка попапа

Пустой HTML-документ с тёмным фоном (`#080b12`), сеткой 40px, инлайн-кнопкой настроек (SVG-шестерёнка) и ссылкой на `panel.js`.

---

### `panel.js` — Движок рендера попапа

| Функция | Назначение | Поведение |
|---------|-----------|-----------|
| `connect()` | Установить порт с service worker | `chrome.runtime.connect({name:'panel'})`. При сообщении: рендерит HTML. При отключении: переподключается через 1 с. |
| `createSettingsBtn()` | Кнопка настроек | Создаёт плавающую кнопку 28x28px (backdrop-blur). Открывает `settings.html` в новой вкладке. |
| `renderHTML(html)` | Отобразить полученный HTML | Парсит через `DOMParser`, извлекает `<link>` и `<style>` в head, заменяет `body.innerHTML`, добавляет кнопку настроек, загружает скрипты последовательно. |

### `panel-render.js` — Взаимодействие и карта в попапе

Выполняется внутри отрендеренного контента попапа. Вся UI-логика и Leaflet-карта.

| Компонент | Назначение | Поведение |
|-----------|-----------|-----------|
| DEC/DMS toggle | Переключение формата координат | Переключает десятичные/DMS, обновляет active-класс. |
| `showToast(msg)` | Подтверждение копирования | Показывает toast "copied" на 1.2 с. |
| coord-cell click | Копирование одного значения | Копирует `data-copy` в буфер обмена. |
| hist-item click | Навигация по истории | Открывает Google Maps для выбранной локации. |
| btnCopy click | Копирование пары lat, lng | Копирует "lat, lng" в буфер. Показывает галочку на 1.5 с. |
| btnDownload click | Экспорт в JSON | Скачивает `ov_{lat}_{lng}.json` с координатами, городом, страной. |
| btnShare click | Поделиться / скопировать URL | Использует `navigator.share` или копирует ссылку Google Maps. |
| Particle system | Ambient-анимация | 30 частиц на canvas 340x760, цвет `rgba(0,212,170,opacity)`, 60fps. |
| `drawGlobe()` | 3D-глобус | Каркасный глобус (сетка 30 градусов) с маркером в точке lat/lng. Скрытие невидимых линий. |
| Map init (Leaflet) | Интерактивная карта | Тёмные тайлы CARTO, слой OpenTopoMap (переключение рельефа), кнопки зума, пульсирующий маркер, GeoJSON границы стран с подсветкой при наведении. |
| terrainBtn toggle | Переключение рельефа | Переключает между тёмной картой и топографическими тайлами. Меняет стиль маркера, границ, наложения. |

**Персистентность карты**: `window._ovMap` хранит экземпляр Leaflet. При повторном рендере карта не создаётся заново.

### `server.js` — WebSocket Relay Server

Node.js HTTP+WebSocket сервер. Пересылает HTML от расширения к браузерам телефонов.

| Функция | Назначение | Параметры | Поведение |
|---------|-----------|-----------|-----------|
| `generateQRSvg(text)` | Сгенерировать QR в SVG | текст для кодирования | Библиотека `qrcode`, акцентный цвет (`#00d4aa`), прозрачный фон, центрированный логотип-круг. |
| `getLocalIP()` | Определить LAN IP | нет | Перебирает сетевые интерфейсы, возвращает первый не-internal IPv4. |
| `doHandshake(req, socket)` | WebSocket handshake | HTTP-запрос + сокет | SHA-1 handshake по RFC 6455. |
| `wsFrame(text)` | Собрать WS-фрейм | текст | Строит текстовый фрейм без маскировки (сервер→клиент), кодировка длины 7/16/64 бит. |
| `sendWS(socket, text)` | Отправить WS-фрейм | сокет + текст | Вызывает `wsFrame()`, отправляет. Тихий перехват ошибок. |
| `decodeFrame(buf)` | Декодировать WS-фрейм | буфер | Парсит маскированные фреймы от клиента. Возвращает UTF-8 строку или null. |
| `notifyPhoneCount()` | Оповестить о количестве телефонов | нет | Шлёт `{type:'phone_count', count}` в сокет расширения. |
| `boot()` | Стартовая анимация в терминале | нет | Прогресс-бар → glitch-эффект → QR-код → пульсирующий индикатор. Через 120 с URL скрываются (privacy). |
| `clearAndRedraw()` | Перерисовать терминал | нет | Выводит iface URL, local URL, пульсирующий индикатор. |
| `printRunning()` | Пульсирующий индикатор | нет | Чередует `◉`/`◌` каждые 300 мс. |
| `scrambleUrls()` | Режим приватности | нет | Очищает терминал, скрывает URL, оставляет только пульсирующий индикатор. |

**Эндпоинты сервера**:

| Путь | Метод | Поведение |
|------|-------|-----------|
| `/qr.svg` | GET | SVG QR-код с URL `http://<LAN_IP>:3000` |
| `/leaflet.js`, `.css`, `/panel-render.js` | GET | Статические файлы для браузера телефона |
| `/` (любой другой) | GET | SHELL-страница — минимальный HTML с WS-клиентом и движком рендера |
| `/ws` | WebSocket | Подключение расширения. Принимает `{type:'render',html}` → рассылает всем телефонам |
| любой другой | WebSocket | Подключение телефона. Получает рендеры в реальном времени. Экспоненциальный backoff (1с→8с) |

**SHELL-страница** (браузер телефона):

Встроенный WebSocket-клиент:
1. Подключается к серверу
2. Обрабатывает `{type:'render',html}`
3. Парсит HTML через `DOMParser`
4. Извлекает стили и скрипты
5. Рендерит в DOM страницы
6. Авто-переподключение с экспоненциальной задержкой

## Полный поток данных

```
1. GeoGuessr API отправляет XHR/fetch-ответ
2. xhr_inject.js перехватывает, отправляет в window
3. content.js парсит lat/lng из ответа
4. Запрос к Nominatim reverse geocode
5. Запросы к Wikipedia, Open-Meteo, REST Countries
6. buildHTML() генерирует полный HTML+CSS+JS документ
7. bridge.js пересылает HTML в background.js
8. background.js кеширует, шлёт в панель и/или WS relay
9. panel.js рендерит HTML в окне попапа
10. panel-render.js запускает Leaflet, частицы, глобус
11. server.js рассылает всем подключённым телефонам
```

## Установка

### Расширение Chrome

1. Откройте `chrome://extensions`
2. Включите **Режим разработчика**
3. Нажмите **Загрузить распакованное расширение**
4. Выберите папку `locanex`

### Relay-сервер (опционально, для телефона)

```bash
npm install
node server.js
```

Отсканируйте QR-код в терминале или в настройках расширения.

- Браузер Chrome/Chromium
- Node.js 18+ для relay-сервера
- Оба устройства в одной сети для mobile relay

---

## Поведение попапа

| Условие | PC попап | Mobile relay |
|---------|:--------:|:------------:|
| Relay выключен | Виден | Выкл |
| Сервер недоступен | Виден | Выкл |
| Сервер включён, телефон не подключён | Виден | Ожидание |
| Сервер включён, телефон подключён | Скрыт | Трансляция |
| Телефон отключился | Снова виден | Выкл |

---

## Источники данных

| Сервис | Использование | Эндпоинт |
|--------|--------------|----------|
| GeoGuessr API | Координаты раунда | Перехват XHR/fetch |
| Nominatim | Обратное геокодирование | `reverse?format=json&accept-language=en` |
| Open-Meteo | Погода | `forecast?current=temperature_2m,...` |
| REST Countries | Население | `v3.1/name/{country}?fields=population` |
| Wikipedia | Статья о городе/стране | `page/summary/{query}` |
| CARTO | Тёмные тайлы карты | `basemaps.cartocdn.com/dark_all/...` |
| OpenTopoMap | Рельеф | `tile.opentopomap.org/...` |

## Структура проекта

```
locanex/
├── manifest.json           Манифест Chrome V3
├── background.js           Service worker — роутинг, попап, WS-клиент
├── content.js              MAIN world — перехват API, обогащение, сборка HTML
├── bridge.js               Isolated world — реле событий в background
├── xhr_inject.js           MAIN world — модификация XHR/fetch
├── panel.html              Оболочка окна попапа
├── panel.js                Подключение порта, движок рендера HTML
├── panel-render.js         UI-взаимодействие, Leaflet карта, частицы, глобус
├── settings.html           Страница настроек
├── settings.js             Логика настроек, QR-код
├── server.js               WS relay сервер — терминальный UI, телефоны
├── leaflet.js              Leaflet 1.9.4
├── leaflet.css             Стили Leaflet
├── assets/
│   └── logo.png            Иконка расширения (128x128)
├── package.json            npm: qrcode зависимость
├── package-lock.json
├── .gitignore
├── README.md               Документация на английском
└── README.ru.md            Документация на русском
```
